# シンプルなライフゲーム（Conway's Game of Life）

Python の標準 GUI ライブラリ `tkinter` だけで作った、ライフゲーム
（Conway's Game of Life）のシミュレーターです。
外部ライブラリは一切不要で、セルの世代交代をリアルタイムに観察したり、
マウスで自由に初期状態を描いたりできます。

---

## ライフゲームとは

イギリスの数学者 John Horton Conway が考案した「セル・オートマトン」です。
格子状に並んだセルが、ごく単純なルールに従って世代ごとに「生」と「死」を
繰り返すだけですが、そこから驚くほど複雑で多様なパターンが生まれます。

プレイヤーが操作するゲームというより、初期状態を決めると
あとは自動的に「進化」していく様子を眺めて楽しむシミュレーションです。

---

## 必要なもの

- Python 3.x
- `tkinter`
  - 多くの Python ディストリビューションに標準で同梱されています。
  - Linux で入っていない場合は `sudo apt install python3-tk` などで導入できます。

外部パッケージのインストールは不要です。

---

## 起動方法

```bash
python simple_game_of_life.py
```

実行するとウィンドウが開き、最初から左上にグライダー（移動するパターン）が
配置された状態で自動的にシミュレーションが始まります。

---

## 操作方法

| キー / 操作   | 動作                                       |
| ------------- | ------------------------------------------ |
| `Space`       | 開始 / 一時停止                            |
| 左クリック    | 生きたセルを置く（ドラッグで連続描画）     |
| 右クリック    | セルを消す（ドラッグで連続消去）           |
| `G`           | フィールド上を「進む」グライダーを置く     |
| `R`           | ランダムなフィールドを生成                 |
| `C`           | フィールドをクリア                         |
| `+`           | 速くする（更新間隔を短く）                 |
| `-`           | 遅くする（更新間隔を長く）                 |

> ヒント: 一時停止（`Space`）してからマウスで好きな形を描き、
> 再び `Space` で再生すると、自分で作ったパターンの進化を観察できます。

---

## ルール

各セルは「生」または「死」の状態を持ち、毎世代、周囲8マスの隣接セルの
状態によって次の状態が決まります。

- **生きたセル**
  - 隣接する生きたセルが **2つか3つ** → 生き残る
  - それ以外（少なすぎる／多すぎる） → 死ぬ（過疎・過密）
- **死んだセル**
  - 隣接する生きたセルが **ちょうど3つ** → 誕生する

盤面の端は「壁」として扱い、範囲外のセルは常に死んでいるものとします
（トーラス状にループはしません）。

---

## 画面の情報

ウィンドウ下部のラベルに、現在の状態がリアルタイムで表示されます。

- **世代** … 経過した世代数
- **生きているセル** … 盤面上の生きたセルの総数
- **状態** … 実行中 / 一時停止
- **速度** … 更新間隔（ミリ秒）。値が小さいほど速い

### 主なクラスとメソッド

| 名前                | 役割                                                       |
| ------------------- | ---------------------------------------------------------- |
| `GameOfLife`        | ウィンドウ・盤面・ループを管理するメインクラス             |
| `draw_cell`         | マウス左クリックで生きたセルを置く                         |
| `erase_cell`        | マウス右クリックでセルを消す                               |
| `place_glider`      | 指定位置にグライダーのパターンを配置する                   |
| `count_neighbors`   | あるセルの周囲8マスの生きたセル数を数える                  |
| `next_generation`   | ルールに従って盤面全体を次の世代へ更新する                 |
| `draw_grid`         | 現在の盤面を Canvas に描画し、統計ラベルを更新する         |
| `update`            | 一定間隔で世代を進めて再描画するメインループ               |

## カスタマイズ

`simple_game_of_life.py` の先頭付近にある定数を変えると、見た目や規模を
簡単に調整できます。

```python
CELL_SIZE = 12   # セル1個の大きさ（ピクセル）
COLS = 70        # 横方向のセル数
ROWS = 45        # 縦方向のセル数
```

- セルを大きく見やすくしたい → `CELL_SIZE` を増やす
- もっと広い盤面で遊びたい → `COLS` / `ROWS` を増やす
- 初期速度を変えたい → `__init__` 内の `self.speed_ms`（既定 120ms）を変更

また、`randomize_key` 内の確率（既定 `0.25`）を変えると、`R` で生成される
ランダム盤面の密度を調整できます。

---

## 仕組みのポイント

- 盤面は `0`（死）と `1`（生）の二次元リストで表現しています。
- 1 世代の更新では、元の盤面を読みながら **新しい盤面** を作るため、
  計算途中の状態が次の判定に影響しません。
- `root.after(self.speed_ms, self.update)` を使い、`tkinter` のイベント
  ループ上で一定間隔ごとに自動更新しています。
